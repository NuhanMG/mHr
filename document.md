# Mobitel Pvt Ltd - HR Assistant System Documentation

## 1. Introduction, Scope, Requirements, Stakeholders

### 1.1 Introduction
The Mobitel HR Assistant is an AI-powered Human Resources application designed to assist employees with HR policies, leave procedures, forms downloading, and answering frequently asked questions (FAQs). Leveraging Retrieval-Augmented Generation (RAG) concepts, the system intelligently finds and synthesizes information from various corporate documents.

### 1.2 Scope
The scope of this system encompasses:
- A conversational AI interface for answering HR-related queries.
- Automated retrieval of HR forms, manuals, and policies based on user intent.
- Pre-configured FAQ browsing.
- Leave optimization logic to suggest the best days for taking leave considering public and company holidays.
- Interaction with Large Language Models (LLMs) such as Ollama (Local) or OpenAI.

### 1.3 Requirements
- **Functional**:
  - Process user queries using NLP.
  - Retrieve exact matching FAQ answers or generate answers using RAG.
  - Serve relevant PDF forms and documents for download.
  - Suggest optimized leave planning.
- **Non-Functional**:
  - High availability and quick response times.
  - Secure handling of internal company documents.
  - Configurable UI and system parameters without code changes.

### 1.4 Stakeholders
- **Mobitel Pvt Ltd**: The organization deploying the solution.
- **HR Department**: Administrators who manage the documents (policies, forms, manuals).
- **Employees**: The end-users of the HR Assistant.
- **IT/Admin Team**: Maintainers of the application infrastructure models.

## 3. Detailed Breakdown of Files and Functions

### 3.1 config.py
Centralized configuration settings.
- **Parameters**: LLM_MODEL (e.g., qwen2.5:7b), EMBEDDING_MODEL (
omic-embed-text:latest), CHUNKING_CONFIG (specific sizes/overlaps for policies, forms, faqs).
- **Control Flags**: ACTIVE_LLM_PROVIDER, RERANKING_ENABLED.
- **Query Expansion**: QUERY_EXPANSION_MAP maps informal terms (like "vacation" or "quit") to formal HR vocabulary.

### 3.2 ackend.py
The core engine for processing queries.
- **get_answer()**: The main RAG pipeline function. It handles phases like document retrieval, context generation, and LLM interaction.
- **Smart Document Detection**: ind_matching_files() scans query keywords against known document topics to explicitly include document downloads.
- **Answer Confidence Check**: Validates the LLM's certainty based on provided context.
- **File Extraction**: Uses regex ([[FILE:...]]) to extract file download paths generated by the AI.

### 3.3 rontend.py
The UI powered by Gradio.
- **ot_turn() & user_turn()**: Handlers for standard chat flow. Update the Gradio history and interact with the backend.
- **aq_click()**: Intercepts clicks on the FAQ accordion to send pre-defined question templates directly to the backend.

### 3.4 aq_data.py
Contains hardcoded QA sets for exact-match bypasses.
- **search_faqs()**: Lexical search script that compares the user's string to predefined questions (e.g., Medical Insurance, Family Packages) and returns a fast response without invoking the LLM if a threshold is met.

### 3.5 leave_optimizer.py & holiday_parser.py
Modules designed to maximize employee time off.
- **holiday_parser.py**: Reads holidays.json to create a list of company/public off-days.
- **leave_optimizer.py**: Given a set of requested days or a month, calculates the strategic combination of annual leave days with public holidays/weekends.

### 3.6 pi_server.py
(Optional/Microservice) Exposes the ackend.py functions via HTTP endpoints (FastAPI/Flask) allowing other corporate portals to integrate the chatbot directly.

## 5. UI and Frontend Integration

The User Interface is built using **Gradio**, providing a clean, responsive web view.
- **Tabs**: Divided into "Chat" and "FAQs".
- **Chat Interface**: A gr.Chatbot component handles the multi-turn conversational history. Below it, a gr.File component dynamically becomes visible if the backend returns physical forms (e.g., PDF applications) matching the intent.
- **State Management**: Uses gr.State initialized with a UUID to track unique user sessions and conversation context, which is mapped back to LangChain's conversational memory formats using convert_gradio_history_to_langchain().

---

## 6. APIs and Context (LLMs & Embeddings)

The system abstracts the Large Language Model provider logic.
- **Ollama**: Represents the on-premise/local privacy-centric AI option. Models like qwen2.5:7b process requests internally without internet traversal.
- **OpenAI**: An optional fallback configured via API Keys for cloud processing.
- The interaction consists of feeding the retrieved context (documents) combined with the user's conversational history into the LLM API using prompt templates, then parsing the returned string.

---

## 7. Vectorstores and Data Processing

Document digestion happens prior to runtime:
- **Chunking**: Governed by CHUNKING_CONFIG in config.py. Forms use tight chunks (500 chars) while policies use broader ones (1200 chars).
- **Embeddings**: Text segments are transformed into dense vector arrays via models like 
omic-embed-text.
- **Storage**: Chunks are stored in a local vector database. 
- **Retrieval**: Uses **MMR** (Maximal Marginal Relevance) to retrieve diverse concepts. The config sets etch_k = 20 and returns k = 8, utilizing lambda_mult = 0.7 to balance relevance with diversity.

---

## 8. Configs & Utils

The config.py acts as the single source of truth:
- **CHUNKING_CONFIG**: Allows granular adjustments to token windows depending on document category.
- **QUERY_EXPANSION_MAP**: Enhances poor user prompts. If a user types "money early", it converts the internal lookup to "salary advance".
- **RERANKING_ENABLED**: Toggles whether initial vector results are passed through a cross-encoder (like ms-marco-MiniLM-L-6-v2) to logically re-sort the findings based on semantic question-answer alignment rather than pure cosine similarity.

## 9. Conclusion

The Mobitel HR Assistant is a comprehensive, production-ready Generative AI enterprise application. By marrying classical exact-match methods (FAQs, strict document fetching) with modern LLM reasoning (RAG, leave optimization), it provides an exceptionally high reliability and utility rate for Mobitel's human resources operations.
